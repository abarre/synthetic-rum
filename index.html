Monitoring

Why care about website performance ?

Two predominant end-user monitoring techniques:


  • Active monitoring (synthetic)
  • Passive monitoring (RUM)

Website performance has a direct impact on every metric you care about.

  • Traffic
  • Conversions
  • Revenue
  • Customer satisfaction
  • Returning visits
Synthetic

Active monitoring


  • Simulated health-checks of the system and its parts
  • Scripted scenarii through an application
  • "Creation" of a single or set of events and/or actions
  • Consistent “clean room” baseline
  • Nothing to install
  • Synthetic is a form of active monitoring
synthetic

How it works

The testing is run on servers in datacenters using a throttled connection to try and mimic the conditions an average user might experience. Pages are loaded in real web browsers and performance metrics are collected.

synthetic

Speed Index

It works by measuring how much of the above-the-fold content is visually complete over time until it is 100% complete. Smaller score is better.
Best correlation to the user exeperience (render time).

RUM

Passive Monitoring




  • Listening to actual traffic as it moves through a system
  • Always on
  • Every user, every browser, every network, anywhere in the world
  • Ability to capture human behavior/events with performance data
  • Only getting better (waterfalls)
  • RUM is a form of passive monitoring
RUM

How it works

A javascript is injected on each page and reports the page load data for every request that is made for each page.
There’s no need to pre-define your important use cases.

Pitfalls

Data are hardly comparable between tools

Synthetic : take the median (of the SpeedIndex) of 10 runs

RUM : remove all the noise that comes with the measure

Pitfalls

Data are hardly comparable between tools

Steve Souders compared the data from a RUM tool and Webpagetest on his website.

Chrome 23 Firefox 16 IE 9
Synthetic First View (secs) 4.64 4.18 4.56
Synthetic Repeat View (secs) 2.08 2.42 1.86
Synthetic 50/50 (secs) 3.36 3.30 3.21
RUM (secs) 9.94 8.59 6.67
RUM data points 94 603 89

Conclusion : be cautious about only showing your synthetic page load times – the real user experience is likely to be quite a bit slower.

Note : At the end of the day the two sources of numbers are different and have very different purposes. Neither of them is wrong or right – they are just different tools.
